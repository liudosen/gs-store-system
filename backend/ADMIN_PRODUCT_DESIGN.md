# 国膳甄选管理后台产品与功能设计

## 背景

国膳甄选管理后台服务于福利商城运营、订单处理、用户资料维护、订阅充值核查和系统运维。后台接口已由 Rust 服务提供，接口前缀主要为 `/api/admin/*`，认证接口为 `/auth/*`。

## 信息架构

一级导航建议如下：

| 导航 | 页面 | 权限码 |
|---|---|---|
| 首页概览 | 经营数据、趋势、最新订单 | `dashboard:view` |
| 商品与分类 | 商品管理、分类管理 | `product:list:view`, `category:list:view` |
| 权益商品 | 权益商品、规格、SKU、库存 | `goods:view` |
| 订单管理 | 订单列表、订单详情、状态处理 | `order:list:view` |
| 用户管理 | 微信用户、实名资料、支付密码查看 | `wechat-user:list:view`, `wechat-user:password:view` |
| 订阅与充值 | 用户余额、余额流水、自动充值、订阅记录 | `subscription:view`, `subscription:record:view` |
| 系统管理 | 管理员授权、日志中心 | `admin:user:view`, `logs:view` |

## 登录页设计

目标：让管理员快速、安全地进入后台，并明确当前连接的服务状态。

功能要求：

| 功能 | 说明 |
|---|---|
| 账号密码登录 | 调用 `POST /auth/login`，提交 `username`、`password` |
| token 存储 | 登录成功后保存 `access_token` 到 `localStorage.guoshan_admin_token` |
| 权限加载 | 登录成功后调用 `GET /auth/codes` 获取用户、角色、权限码 |
| 回跳 | 支持 `redirect` 参数，未登录访问后台页后登录成功回原页面 |
| 错误反馈 | 登录失败展示明确错误，不吞掉后端 message |
| 加载状态 | 提交期间禁用按钮，显示 loading |
| 可访问性 | 表单有 label，按钮可键盘提交，焦点态清晰 |

## 接口契约

### 登录

`POST /auth/login`

请求：

```json
{
  "username": "admin",
  "password": "******"
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "access_token": "jwt",
    "expires_in": 86400,
    "token_type": "Bearer"
  },
  "message": "success"
}
```

### 权限码

`GET /auth/codes`

请求头：

```text
Authorization: Bearer <token>
```

成功响应包含：`username`、`role`、`is_admin`、`codes`。

## UI 设计原则

| 维度 | 方案 |
|---|---|
| 风格 | 专业运营后台，清爽、高信息密度、弱装饰 |
| 布局 | 桌面优先，左侧导航 + 顶部工具栏 + 内容卡片 |
| 色彩 | 中性浅色底，蓝绿品牌色作为主行动色 |
| 表格 | 支持筛选、分页、空状态、错误状态、加载骨架 |
| 表单 | label 常驻、错误靠近字段、提交 loading |
| 危险操作 | 删除、状态变更、自动充值、查看支付密码均二次确认 |
| 响应式 | 桌面表格，窄屏折叠为卡片/抽屉导航 |
| 可访问性 | 对比度达标，焦点态可见，不只靠颜色表达状态 |

## 实施优先级

| 优先级 | 页面 |
|---|---|
| P0 | 登录、首页概览、商品管理、权益商品管理、订单管理、用户管理 |
| P1 | 分类管理、用户余额流水、订阅记录、管理员授权 |
| P2 | 日志中心、上传体验增强、更多统计图表 |
