# 公司官网 MVP 规划

## 1. 背景与目标

当前项目的 MVP 需要把官网和业务产品入口分开：

- 用户直接访问 `www.mcx59481.cn` 时，首先看到的是公司官网
- 官网承担品牌介绍、信任建立、社会价值表达、合作展示、退役军人入驻转化
- 真正的业务产品入口从官网进入，而不是直接把用户丢进下单系统

这意味着当前域名首页的首要任务不是交易，而是：

- 建立公司可信度
- 解释公司在做什么
- 让老人家庭、合作方、退役军人理解平台价值
- 提供明确的“退役军人入驻”入口

## 2. 角色协同结论

### 2.1 产品经理视角

需要先明确品牌定位：

- `迷彩侠（mcx）` 不是单纯的养老服务公司
- `迷彩侠（mcx）` 的核心是一个依托退役军人群体创造社会价值的平台
- 养老上门服务是当前 MVP 的主要落地场景之一，不是品牌的全部边界
- 平台真正表达的是：帮助退役军人就业、组织退役军人服务社会、让社会看见退役军人的持续价值

官网的核心不是“展示所有功能”，而是完成 4 个任务：

1. 让第一次访问的人在 5 秒内知道平台在做什么
2. 让用户理解平台如何以退役军人为核心创造社会价值
3. 让访问者看到真实可信的服务成果、组织能力和老兵就业价值
4. 让退役军人快速找到入驻入口并产生行动

### 2.2 UI 视角

根据 `ui-ux-pro-max` 的规则，官网应遵守：

- 移动端优先
- 强信息层级
- 单页滚动结构清晰
- 一个页面只保留一个主转化目标
- 不依赖悬停交互
- 按钮和触控区域足够大
- 不要做花哨但削弱信任感的视觉风格

适合本项目的视觉方向：

- 可信、温暖、克制
- 具有公益感但不“苦情化”
- 兼顾军人气质、组织感、社会责任感与社区服务的柔和感

不建议：

- 赛博感
- 过强科技蓝紫渐变
- 过度卡片堆砌
- 营销感过重的夸张 hero

### 2.3 前端视角

官网应当按“内容官网”来做，而不是把它当后台或商城首页：

- 单独官网路由或单独官网首页
- 组件化 section 结构
- 支持后续替换真实文案、数据、图片
- 支持内容块动态化配置
- PC 端不做业务系统，但官网本身建议可响应式展示

说明：
- “PC 不支持”是指业务产品不支持 PC 操作
- 公司官网本身仍建议具备基础响应式，否则品牌传播和对外访问会很受限

## 3. 官网 MVP 的核心任务

官网 MVP 只解决三件事：

1. 介绍公司
2. 建立信任
3. 引导转化

其中转化优先级排序建议：

1. 退役军人入驻
2. 合作方理解平台模式并建立合作意向
3. 用户了解当前服务场景

## 4. 官网信息架构

建议官网首页采用单页长滚动结构，移动端优先展示，按下面顺序组织：

### 4.1 顶部导航

建议包含：

- 品牌 Logo / 公司名
- 首页
- 关于我们
- 服务内容
- 最新动态
- 老兵培训
- 联系我们
- 退役军人入驻按钮

移动端导航建议：

- 顶部简洁导航栏
- 右上角折叠菜单
- `退役军人入驻` 按钮保持高优先级

### 4.2 首屏 Hero

首屏必须立刻回答三个问题：

- 这是什么平台
- 它依托谁、为社会解决什么问题
- 用户下一步做什么

建议首屏内容结构：

- 主标题：一句话表达平台使命
- 副标题：解释“以退役军人为核心，连接就业、服务与社会价值”的平台定位
- 主按钮：`退役军人入驻`
- 次按钮：`了解平台`
- 配图：真实服务场景或组织活动场景

说明：
- 首屏不需要塞过多数据
- 更重要的是情绪、可信度、方向感

### 4.3 公司介绍板块

目标：

- 用简洁方式说明平台是谁、为什么做这件事

建议内容：

- 公司简介
- 成立初心
- 平台理念
- 平台使命

适合的表达方式：

- 左文右图
- 或一段简洁介绍 + 3 个价值关键词

### 4.4 我们在做什么板块

目标：

- 让用户快速理解平台如何组织退役军人创造社会价值

建议内容：

- 退役军人就业赋能
- 社区与家庭服务落地
- 当前 MVP 通过陪诊、陪伴、家政、送餐等服务连接老人家庭与退役军人

建议展示形式：

- 3 步流程图
- 或 4 张场景卡片

### 4.5 服务场景板块

目标：

- 展示平台覆盖的典型服务类型

建议板块内容：

- 陪诊服务
- 生活陪伴
- 家政清洁
- 送餐到家
- 代买代办
- 节日关怀 / 社区活动支持

建议形式：

- 图标 + 标题 + 一句话说明

### 4.6 数据成果板块

目标：

- 建立信任感

这个板块很重要，因为你已经明确提到“服务的老人数据介绍板块”。

建议展示指标：

- 已入驻退役军人数量
- 已服务老人数量
- 服务覆盖城市/社区数量
- 累计订单量
- 退役军人就业带动人数
- 用户满意度

注意：

- 当前文案和真实数据未定时，先用占位结构
- 不建议放假大空数字

### 4.7 公司活动板块

目标：

- 让用户看到这不是一个冷冰冰的平台，而是真正有组织行动力的公司

建议内容：

- 公益活动
- 社区走访
- 敬老主题活动
- 退役军人志愿服务活动
- 就业赋能与联建活动

建议形式：

- 图片列表
- 时间轴
- 活动卡片

### 4.8 最新动态板块

目标：

- 体现公司持续运营中，不是静态项目页

建议内容：

- 公司新闻
- 项目进展
- 合作发布
- 媒体报道
- 老兵就业与培训动态

建议形式：

- 3 到 6 条卡片式动态
- 支持跳转详情页

### 4.9 老兵培训板块

目标：

- 增强平台的组织能力表达和退役军人就业转化

建议内容：

- 培训内容方向
- 服务规范
- 礼仪与沟通
- 陪诊与安全常识
- 平台入驻培训流程

这个板块的作用不仅是展示培训，还在于传达：

- 平台不是随意派人
- 平台有标准化培训和管理机制
- 平台在持续为退役军人提供职业路径与社会服务能力建设

### 4.10 退役军人入驻招募板块

这是官网最重要的转化模块之一。

建议内容：

- 为什么加入平台
- 平台能提供什么
- 入驻基本条件
- 入驻流程
- 入驻按钮

建议 CTA：

- `立即申请入驻`
- `查看入驻条件`

### 4.11 合作与信任板块

目标：

- 面向政府、社区、养老机构、公益组织、企业、媒体等建立合作信号

建议内容：

- 合作单位 Logo
- 战略合作方向
- 平台保障机制

### 4.12 联系我们板块

建议包含：

- 公司名称
- 联系电话
- 邮箱
- 公司地址
- 微信公众号/客服二维码

### 4.13 页脚

建议包含：

- 公司版权信息
- 备案信息
- 隐私政策
- 用户协议
- 联系方式

## 5. 官网首页推荐顺序

移动端官网首页建议顺序如下：

1. 顶部导航
2. 首屏 Hero
3. 公司介绍
4. 我们在做什么
5. 服务场景
6. 数据成果
7. 公司活动
8. 最新动态
9. 老兵培训
10. 退役军人入驻
11. 合作与信任
12. 联系我们
13. 页脚

这个顺序的逻辑是：

- 先讲使命
- 再讲业务
- 再讲成果
- 再讲组织能力
- 最后做转化

## 6. 官网 UI 设计方向

### 6.1 品牌气质

建议关键词：

- 温暖
- 可靠
- 正直
- 有行动力
- 有社会责任感

### 6.2 视觉风格

建议采用：

- 现代简洁
- 轻内容型品牌官网
- 图片驱动 + 数据增强
- 大面积留白 + 节制配色

不建议：

- 紫色科技感
- 过度玻璃拟态
- 炫光渐变主导
- 复杂插画堆满首页

### 6.3 色彩方向

建议主色逻辑：

- 主色：沉稳蓝 / 深青蓝
- 辅色：暖橙 / 暖金
- 中性色：米白、浅灰、深灰

原因：

- 蓝色承载信任与组织感
- 暖色承载养老服务的人情味
- 避免页面过冷

### 6.4 字体与排版

建议：

- 标题清晰有力量
- 正文高可读性
- 移动端默认 16px 起
- 重点数字用更强字重

### 6.5 图片策略

优先使用：

- 真实活动照片
- 服务现场照片
- 培训现场照片
- 团队工作照片

避免：

- 过度摆拍的图库图
- 模糊抽象背景图
- 与业务无关的“温馨家庭样板图”泛滥

## 7. 官网前端实现建议

### 7.1 页面结构

前端建议拆为以下 section 组件：

- `HeroSection`
- `AboutSection`
- `HowItWorksSection`
- `ServiceSceneSection`
- `ImpactStatsSection`
- `ActivitiesSection`
- `NewsSection`
- `TrainingSection`
- `VeteranRecruitSection`
- `PartnersSection`
- `ContactSection`

### 7.2 内容配置化

因为你说文案还在准备中，所以官网应按“结构先行、内容后填”方式实现。

建议把以下内容配置化：

- 标题
- 副标题
- 板块描述
- 数据指标
- 活动卡片
- 动态列表
- 培训内容列表
- CTA 文案

### 7.3 路由策略

建议：

- `/` 为公司官网首页
- `/veteran-join` 为退役军人入驻页
- `/news` 和 `/news/:slug` 为动态页
- 后续业务系统可用单独路由承载

## 8. MVP 非目标

官网 MVP 先不做：

- 复杂内容后台
- 多语言
- 视频直播
- 重交互动画
- 复杂地图与数据大屏
- 在线客服系统深度集成

## 9. 后续文案与素材准备清单

为了把官网真正做出来，后续需要补这些内容：

### 9.1 文案

- 公司简介
- 使命愿景
- 服务介绍
- 活动介绍
- 培训介绍
- 入驻说明
- 联系方式

### 9.2 数据

- 服务老人数量
- 入驻老兵数量
- 累计服务次数
- 活动次数
- 覆盖区域

### 9.3 素材

- Logo
- 品牌色
- 活动照片
- 服务照片
- 培训照片
- 合作单位 Logo

## 10. 最终建议

对于当前 MVP，官网不要做成一个“功能入口集合页”，而应做成一个“可信的平台品牌首页 + 清晰的老兵入驻转化页”。

一句话概括：

- 官网先讲清楚“我们是谁、我们依托谁创造价值、我们做成了什么、为什么值得信任”
- 再用明确的 `退役军人入驻` 按钮完成首个关键转化
